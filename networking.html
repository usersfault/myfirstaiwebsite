📘 What is Networking?

Networking is the backbone of the internet. It’s how computers, servers, devices, and services talk to each other. Everything from websites to hacking tools depends on understanding networking.

🧱 OSI Model (7 Layers)

This is a conceptual model that breaks networking into 7 layers:

  1. Application – HTTP, FTP, DNS
  2. Presentation – Encryption, compression
  3. Session – Opening, managing, and closing sessions
  4. Transport – TCP/UDP, ports
  5. Network – IP addressing and routing
  6. Data Link – MAC addresses, Ethernet
  7. Physical – Cables, switches

🧰 Tools I’ve Learned

  • Nmap – Scans for open ports and services
  • Netcat (nc) – Create TCP/UDP connections and backdoors
  • Telnet – Connect to ports and test services
  • OpenSSL s_client – Test HTTPS and SSL connections
  • Dig – Look up DNS records (e.g. A, MX, TXT)

🔗 Protocols I Studied

  • HTTP – Stateless protocol used by web browsers
  • HTTPS – HTTP with encryption via SSL/TLS
  • TCP – Connection-based, reliable
  • UDP – Connectionless, faster, used in streaming
  • DNS – Resolves domain names to IP addresses
  • SSH – Secure remote login over a network

🔐 Concepts I Explored

  • Proxies – Middleman between user and internet (can hide your IP)
  • VPN – Encrypts traffic and hides your location
  • Tor – Bounces traffic through multiple relays to ensure anonymity
  • Port Scanning – Checking which services are running on a server
  • IP Addressing – Public vs private IPs

🧠 Insights & Realizations

  • Everything online runs on ports and protocols
  • Knowing how TCP/IP and HTTP work helps in bug bounty
  • Proxies and VPNs don’t make you fully anonymous
  • You can test websites without a browser using tools like curl, telnet, or netcat
