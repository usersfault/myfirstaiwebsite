🧠 Why Linux?

Linux is the base of most servers, hacking tools, and open-source systems. It’s flexible, powerful, and made for people who love control. For Ayush, Linux became the first real step into hacking, networking, and building smart systems.

📂 Useful Linux Commands

  • ls – List files and directories
  • cd – Change directory
  • chmod – Change file permissions
  • chown – Change file ownership
  • find – Search files by name, type, or permission
  • cat – Display file contents
  • scp – Secure file copy over SSH
  • sudo – Run commands as root
  • history – Show previously executed commands

🔐 File Permissions & SetUID

Ayush explored the power of Linux permissions using:

  • chmod u+s file – SetUID bit, lets users run the file as the owner (commonly root)
  • ls -l – Shows permission bits like -rwsr-xr-x
  • chmod u-s file – Remove the SetUID bit
  • Note: You may see “Operation not permitted” if you try removing SetUID without root access

🧱 System Hardening (Security Basics)

Ayush learned that hardening means reducing vulnerabilities. On Linux, that includes:

  • Disabling unused services
  • Setting file permissions strictly
  • Using tools like ufw or iptables to control traffic
  • Installing fail2ban to prevent brute force attacks
  • Regular system updates and kernel patches
